Sistema Contable Completo

Desarrollé un sistema de gestión de comprobantes (facturación) modular y adaptable, construido íntegramente en PHP. El objetivo principal fue crear una aplicación robusta que pudiera funcionar en diferentes entornos con una configuración mínima.

El proyecto demuestra habilidades en arquitectura de software, manejo de bases de datos y simulación de servicios externos para pruebas.

1. Arquitectura de Datos Flexible (Estrategia de Fallback)

El mayor desafío fue asegurar que el sistema siempre funcionara, sin importar la configuración del servidor del usuario. Para lograrlo, implementé una estrategia de persistencia de datos con un "patrón de fallback" de tres niveles:

Nivel 1: MySQL (Opción Recomendada)

  • Es la opción principal y más robusta.
  • Creé un script de instalación (conectar_bd.php) que automatiza todo el proceso: verifica la conexión al servidor, crea la base de datos sistema_comprobantes si no existe, y ejecuta el script mysql_setup.sql para generar la estructura de tablas.

Nivel 2: SQLite (Respaldo Automático)

  • Si el sistema detecta que la extensión PDO para MySQL no está disponible o la conexión falla, automáticamente utiliza SQLite.
  • Esto permite que la aplicación funcione sin necesidad de un servidor de base de datos externo, creando un archivo database.sqlite local.

Nivel 3: Archivos JSON (Fallback Definitivo)

  • Como última instancia, si ni MySQL ni SQLite están disponibles, el sistema entra en modo "fallback" y guarda todos los datos en archivos .json en la carpeta data/.

2. Simulación de API Externa (AFIP)

Para desarrollar y probar la funcionalidad de facturación electrónica sin depender de una conexión real a la AFIP, desarrollé un módulo de simulación que:

  • Simula respuestas del servicio web de la AFIP
  • Genera datos ficticios pero realistas, incluyendo CAE y códigos QR
  • Incluye simulación de errores comunes para testing robusto

Landing Page Corporativa "Hojalatería Chartier"

Desarrollé una landing page corporativa One-Page (de una sola página) para "Hojalatería Chartier", un proyecto enfocado en crear una presencia digital moderna, profesional y 100% responsiva.

El principal objetivo fue construir una interfaz de usuario atractiva y fluida utilizando únicamente tecnologías web puras (HTML, CSS y JavaScript "Vanilla"), sin depender de frameworks o librerías externas.

1. Maquetación Avanzada con CSS Grid y Flexbox

El proyecto demuestra un dominio profundo de las técnicas de layout modernas de CSS para crear un diseño adaptable a cualquier dispositivo, desde móviles hasta monitores de escritorio.

  • Diseño Responsivo (Mobile-First): La estructura se diseñó desde cero para ser "mobile-first". Utilicé Media Queries de forma intensiva para reorganizar el contenido, asegurando una experiencia de usuario óptima en tablets y móviles.
  • CSS Grid: Implementé CSS Grid para la sección de "Contacto", creando una maquetación asimétrica y compleja que posiciona las tarjetas de información y el formulario de manera organizada y visualmente impactante.
  • Flexbox: Usé Flexbox en todo el sitio para el alineamiento de componentes, como la barra de navegación, el hero section y las tarjetas de servicios.

2. Interactividad con JavaScript Puro (Vanilla JS)

Toda la interactividad del sitio se programó con JavaScript "Vanilla" (puro), demostrando una sólida comprensión de la manipulación del DOM.

  • Navegación Responsiva: Programé el menú de navegación móvil (tipo "hamburguesa"), que se desliza desde el lateral y aplica un overlay al contenido principal, todo manejado con eventos de click y conmutación de clases CSS.
  • Acordeón de Servicios: Desarrollé la funcionalidad "VER MÁS / VER MENOS" de las secciones "Servicios" y "Productos". Mediante JavaScript, el botón detecta el estado de la tarjeta y añade o quita una clase (.expanded), que a su vez activa una animación CSS fluida que expande o contrae el contenido.
  • Formulario de Contacto (Client-Side): El formulario de contacto se maneja 100% del lado del cliente. Un script captura el evento submit, previene la recarga de la página, y utiliza los datos ingresados para generar dinámicamente un enlace mailto: con la información pre-cargada.
